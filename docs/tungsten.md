# Tungsten Programming Language

## Overview
Tungsten is a high-level dataflow programming system designed for the Cerebras Wafer-Scale Engine cluster. Its primary purpose is to bridge the gap between high-level domain descriptions (specifically for scientific computing and physics simulations) and the low-level dataflow representation required by the hardware.

## Core Concept: Dataflow Domain Translation
Tungsten implements a "Dataflow Domain Translation" approach. Rather than requiring the programmer to manually map every operation to the 2D mesh of cores, Tungsten translates domain-specific mathematical descriptions into a dataflow graph.

### Key Areas of Application
- **Stencil Computations:** Optimized for operations where a value is computed from a fixed pattern of neighboring values.
- **Finite Difference Methods:** Specifically designed for solving partial differential equations (PDEs).
- **Scientific Simulations:** Demonstrated effectiveness in simulating Shallow Water Equations.

## Execution Model
Tungsten leverages the unique architecture of the Cerebras cluster by:
1. **Mapping Domain to Hardware:** Translating mathematical operators directly into the spatial layout of the WSE.
2. **Distributed Processing:** Utilizing the massive parallelism of the wafer-scale engine to achieve "Beyond Exascale" performance.
3. **Data-Driven Execution:** Following the dataflow paradigm where computation is triggered by the availability of input data from the fabric.

## Relation to CS3 Simulator
For the purpose of this simulator, Tungsten represents the high-level "compiler" or "orchestrator" layer. The simulator must be able to:
- Execute the dataflow graphs generated by Tungsten.
- Model the stencil patterns that Tungsten optimizes for.
- Simulate the efficient time-integration and data movement patterns used in Tungsten-translated workloads.
